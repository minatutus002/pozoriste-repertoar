# Pravila za doprinos

1. Forkuj repozitorijum i kreiraj branch iz `develop` ili `main`:
   - feature/<opis>
   - fix/<opis>

2. Commit poruke:
   - Koristi kratak prefiks i opis: `feat:`, `fix:`, `chore:`, `docs:`
   - Npr: `feat: dodaj rezervaciju sa validacijom`

3. Pokreni testove i proveri da li se build uspešno izvršava pre PR-a.

4. Otvori Pull Request sa opisom i spiskom promena.

5. Oèekuj review i ispravke pre merge-a.

6. Velike izmene i izmene koje utièu na DB schema treba najaviti u Issues pre implementacije.
